DESCRIPCION
En los listados no deja que el motor genere la sentencia SQL sino la indicada por nosotros. Con esta etiqueta se para la sentencia sql del formulario y se envía al motor de Base de datos la creada posteriormente por el usuario.  Ya que el motor no genera el cursor habrá que asignar el número total de registros a extraer a la variable "$_TReg", a no ser que se genere la variable "$usuCursor".
Se puede ejecutar solo la setenecia sql y dejar al motor que extraiga la información si esta no hay que tratarla.
Hay que tener en cuenta en los listados paginados solo se entra en esta etiqueta en la primera página y si modificamos desde aquí alguna variable del sistema no se tendrá en cuenta en las siguientes paginaciones.
Hay que tener encuenta que el motor muestra las condiciones con las que se ha creado un listado de la última sentencia sql que se halla ejecutado en esta etiqueta, si esa sentencia no fuera la que indicase la condición se rellenaria la variable "$_SQL_" que es donde se guarda la última sentencia sql.
Si se quiere que el sistema ejecute un sql se le asignará a la variable "$_SQLExecute",de esta variable se deducirá el "count" para saber los registros afectados, si se quiere hacer el "count" será mediante la variable "$_CountExecute", esta dos variables contendrán el sql a ejecutar.
Si se quiere sacar un listado sin registros hay que definir la variable "$EmptyList = true;".
