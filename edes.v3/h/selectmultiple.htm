SelectMultiple
SINTAXIS
[SelectMultiple] a,?R | Field |  LongitudDelCampo[,maxItem] [BOX]  [ | SQL [ | Sort=false [ | minHeight[,maxHeight] ] ] ]
[SelectMultiple]   ?     | ListaFields
DESCRIPCION
Mantenimiento de un campo que se puede seleccionar ,mas de un option alimentado por un select.
Esta etiqueta tiene dos comportamientos en los modos de consultas (c,b,m) se puede seleccionar mas de un opción pulsando la tecla control+click y en el resto de modos es un select que te permite seleccionar mas de un option mostrandote los opction fuera del select pero almacena todas las claves de los option seleccionados en el mismo campo separados los valores por coma y emezando y teminando por coma.
Si se quiere activar el SelectMultiple en todos los select se definirá en el fichero "sql.ini" la variable "$_SelectMultiple = true;"
A estos campos se les puede añadir y eliminar item individuales con los prefijos "+" y "-", S(campo).val("+01") o S(campo).val("-01") poniendo o quitando solo el elemento "01" o con varios S(campo).val("-01,+02,+03")
PARAMETROS
 Mode    Modo/s en la que está activa la etiqueta.
 Field  Nombre del campo
 LongitudDelCampo  Es la longitud del campo donde se guardarán todas las selecciones separadas por coma.
- Si queremos poner un máximo de item pondremos "," y el número máximo de item.
- Si ponemos la partícula "BOX" el [SelectMultiple] tendrá todos los item dentro del campo
 SQL  sql a ejecutar para extraer los elementos a seleccionar.
 Sort  Por defecto no ordena la selección. 
 minHeight,MaxHeight  Por defecto la altura de lo seleccionado es el de su contenido, pero se puede poner una altura mínima y si se quiere condicionar la altura máxima. 
EJEMPLO
[SelectMultiple] * | multi_provincia | 12
[Fields]
    Multiples provincias | multi_provincia:cd_prov | X | S | 40 || M |||
El select se rellena de una tabla
