Debug
SINTAXIS
[Debug] Mode | Type [ [ | Usuario ]  | LabelARastrear ]
DESCRIPCION
Ejecutar el archivo de definición en modo debug, en el grupo de fichas sólo está disponible en el GDF.
PARAMETROS
 Mode  
 Type Tipo de debug (ver tabla).  
 Usuario  Si se pone el nº de usuario la etiqueta solo tendrá efecto para ese usuario.
En el DF se puede poner indistintamente el valor numérico o el literal, pero si se quiere modificar el valor del Debug dentro de código PHP se tiene que utilizar el valor numérico.
 LabelARastrear Cuando "Type==TMP" admite este parámetro, pudiendo se este el nombre de la etiqueta a rastrear o "JS" para indicar todos los javascript,  que en las etiquetas multilinea se introducirá código para detectar donde se produce el error mediante la variables "__eLINE__" que se le asignará un valor en cada linea, sabiendo en caso de error con que valor quedó la variable, le añadimos el parámetro ",console" en lugar de generar la variable de rastreo generará la instrucción "console.log(...Código...)".

Los posibles valores para el parámetro Type son:
0 OFF Desactivado
1 SQL Se imprimen todas las salidas SQL
2 TMP No se borran del subdirectorio temporal los includes PHP del usuario como son "DBIni" y "PHPForm", etc.
3 BORDER Pinta los bordes de las celdas del formulario
4 SaveSQL Se graban todas las sentencias SQL en el modo indicado.
5 InOut Se mostrará una tabla con la lista de campos con el valor de entrada y el que se grabaría en la DB en las operaciones de Alta y Modificación, sin llegar a modificarla.
6 WInOut Igual que el 5 pero se abre en una ventana nueva descargando la página.
7 NoZip Se desactiva la compresión de la página, aveces no se puede ver el código de la página con la compresión activa.
8 Flag En las etiquetas multilinea de JavaScript se mostrará un "alert()" indicando inicio y fin del código.
9 DF Mostrará el código del DF ( Definition File ) a ejecutar, al poner este parámetro la etiqueta se debe de poner al principio del script o ponerlo temporalmente en "sql.ini".
10 SaveSQLH Igual que "SaveSQL" pero mantiene el historial
11 SaveDF Grabará en gsTron el código del DF ( Definition File ) a ejecutar, al poner este parámetro la etiqueta se debe de poner al principio del script o ponerlo temporalmente en "sql.ini".
12 phpInfo  Mostrará sólo el resultado de la función "phpinfo()".
13 ViewFields Mostrará una ventana con los campos antes de enviar al servidor en el alta y en la modificación.
14 Eval (futuro valor)
15 Observe Hace saltar el debug al modificar el vlue de un campo, una propiedad o un atributo del style. Con este parámetro esta etiqueta se puede repetir.
20 SelInfo Se rastrea todas las operaciones de la etiqueta [SELINFO].
30 ErrSQL Las funciones SQL devolverán el valor -1si se produce algún error y no se parará la ejecución.
31 HTM Se graban las páginas a enviar al pc en "/_tmp/log/[_User]_ddHHiiss.htm" ( date('dHis') ).
  DATA Muestra la información de las matrices $_POST, $_GET y $_FILE.
EJEMPLO
 
 
EJEMPLO
[Debug] mR | observe || :cd_banco                     // al modificar el campo
[Debug] mR | observe || :cd_banco | style,leng        // al cambiar el style o el attribute "Leng"
[Debug] mR | observe || :cd_banco_grupo               // al modificar el campo
El parámetro "4" identifica al objeto (":" fields, "# "id, "." class) y el "5" parámetro "style" y/o cualquier nombre de attribute.
