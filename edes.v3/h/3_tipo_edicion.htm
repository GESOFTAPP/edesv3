3. Tipo edición
DESCRIPCION
Indica qué tipo de control de edición ha de aplicarse al campo para que el usuario sólo pueda introducir determinados patrones de caracteres.

Los números ni las fechas admiten el "*" en las búsquedas.

Los posibles filtros de control de edición son:

T  Teléfono, obligatorio 9 caracteres.

 Si está activo el 3CX aparecerá un icono a la derecha del campo para llamar por teléfono, teniendo el usuario la posibilidad   de cambiar el número añadir un prefíjo variable mediante la función de usuario "FUNCTION_phone()".
          function FUNCTION_phone(tlf){
               return false;
               return tlf;
               return newtlf";
          }
0 (cero) Sólo números (tipo string)
DNI DNI
NIF  Letra del NIF si la longitud es de 1 y el NIF completo si su longitud es de 9.
nif  Acepta tanto DNI como NIF de 9, si es un DNI formateará con ceros por la izquierda.
CIF  CIF de empresa
cif  CIF de empresa, se chequea sólo que los 7 carácteres del centro sean números.
CP  Código Postal
N  Nombre en mayúsculas
n  Nombre en minúsculas
#N  Nombre con mayúsculas y minúsculas
D  Dirección en mayúsculas
d  Dirección en minúsculas
#D  Dirección con mayúsculas y minúsculas
f  Nombre de fichero en minúsculas
F  Nombre de fichero en mayúsculas
CDI  Fecha y hora en formato AAAA-MM-DD HH:MM:SS
F4  Fecha con 4 dígitos para el año
P4  Periodo con 4 dígitos para el año
C  Password / Clave
X  Cualquier cosa en mayúsculas
x  Cualquier cosa en minúsculas
#X  Cualquier cosa en mayúsculas y minúsculas
DC  Dígitos de control de la cuenta corriente
NSS  Número de la Seguridad Social (son 11 caracteres)
TC  Tarjeta de crédito (solo nº, sin desarrollar el chequeo)
H  Hora
@  Dirección de correo
W  Dirección de web, si el nombre del campo es "facebook" o "twitter" pondrá su logo particular.
*  Campo Autoincremental / serial, por tanto no se grabará.
#  Campo de notas, puede contener cualquier contenido, permite retornos de carro (textarea) 
#L  Igual que '#' pero solo minúsculas
#U  Igual que '#' pero solo mayúsculas
+  (signo más)  Nº positivos
-  (signo menos)  Nº positivos y negativos
+,  (signo más seguido de una coma) Nº positivos que pueden contener decimales
-,  (signo menos seguido de una coma) Nº positivos y negativos, ambos pueden contener decimales 
CLR  Tipo de campo para guardar un color hexadecimal (CoLoR)
IP  IP

Si se quiere evitar el filtrar caracteres se puede definir la siguiente etiqueta:
[AddCode] a,mR | field | I | NotFilter=1

Los campos que se quiera que admitan acentos se pondrá la propiedad "eAccent=1"
[AddCode] a,mR | field | I | eAccent=1
PARAMETROS
Los campos numéricos se puede definir el comportamiento del valor "cero" con la variable "_ShowZero"
1.- Se muestra el valor cero.
0.- Si es cero se muestra el campo vacio, valor por defecto.
-1.- Se muestra cero solo si se mete el valor cero, es decir que en un campo numérico si pones cero se verá y si lo borras se verá vacio.
EJEMPLO
[JSIni] *
     _ShowZero = 1;
Si sequiere definir en comportamiento de forma global se definirá en el fichero de configuración "group.var" apartado "Tab" y propiedad "ShowZero".
