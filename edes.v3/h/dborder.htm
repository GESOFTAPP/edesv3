DBOrder
SINTAXIS
[DBOrder] Field [DESC] [, Field [DESC]] ... [, Field [DESC]]
DESCRIPCION
Columnas de ordenación para los listados, en el caso del grupo de fichas (GDF) se debe poner en la primera de ellas.
En caso de incluir el parámetro "DESC" detrás de un campo, la ordenación del mismo será descendente.
El los listados también se puede añadir la cláusula "group by" después de la lista de campos a ordenar.
Si quieres ordenar un listado por un campo select "cd_[tabla]" se tendrá que poner "nm_[tabla]".
Se puede definir la ordenación en petición mediante la variable del sistema "_ORDEN_" declarandola como un select virtual "SV".
PARAMETROS
EJEMPLO
[DBOrder] codigo, fecha desc, articulo
[DBOrder] nm_prov group by 1,2
EJEMPLO

[AddOption] c | _ORDEN_ | refe,Nº Referencia; A.cd_prov+A.cd_muni+A.cd_distrito+A.cd_barrio+A.nombre,Distrito y Barrio; pvp,PVP; pvp DESC, PVP Descendente; L.nombre+K.apellidos,Asesor

[Fields]
Ordenación | _ORDEN_ | 0 | SV | 3 | | Q | | |

Declarando la ordenación mediante un select virtual
EJEMPLO
[DBOrder] B.nm_auto, C.nm_t_cuota, A.dt_ini DESC
En caso que hubiera conflicto con nombres de campos se le puede preceder con el alias de la tabla, teniendo encuenta que los alias de las tablas empiezan por la "A", "B", etc y habrá uno por cada select y en el orden en que se encuentren en el listado.
