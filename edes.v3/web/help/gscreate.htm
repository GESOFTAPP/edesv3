AYUDA PARA LA CONFIGURACION DE LA INTRANET (gsCreate)

AYUDA

  • Instalación Programas

    Fichero de ayuda paso a paso para la instalación y configuración de los programas que integran el sistema para la Intranet. Sistema Operativo, Servidor de Paginas, Lenguaje de scripts, Motor E-DES, Programa de creación del Proyecto "GSCREATE".
  • Definición SQL

    Para la versión Beta del Motor E-DES todavía no está operativo el IDE configurador del proyecto inicial y en el caso de definir el modelo de datos se necesita disponer de la estructura en un fichero plano con una serie de modificaciones para crear al mismo tiempo los formularios de la aplicación. Esta opción muestra como crear ese fichero plano que se incluirá en la opción "Definir Base de Datos del menú Intranet".

ESTADO

  • Php

    Pantalla que muestra la información del estado del servidor de Script PHP
  • Esplorador

    Pantalla que muestra la información de la configuración del navegador del PC local

CONFIGURACION

  • Php.ini

    Pantalla para visualizar el fichero php.ini.
  • Httpd.conf

    Pantalla para visualizar el fichero httpd.conf.

PROYECTO

  • Crear

     Nombre del directorio donde se almacenará el proyecto de Intranet. En dicho directorio se ubicarán los ficheros de configuración y la estructura de la Intranet final
  • Seleccionar

     Elige un proyecto para ver o modificar los datos de configuración.
  • Renombrar

     Cambiar el nombre del proyecto por lo tanto el nombre del directorio donde esta ubicado.
  • Borrar

     Eliminar físicamente del disco el directorio con todos sus datos del proyecto
  • Configurar
     DATOS BÁSICOS DE LA EMPRESA. 
          Empresa: Nombre de la empresa.
      Titulo Aplicación: Titulo principal de la Intranet. (aparece en la Parte superior)
      Subtitulo Aplicación: Subtitulo de la Intranet.(aparece en la Parte superior)
      Dirección Web: Dominio donde reside la Intranet.

     DDBB.
      Base de Datos: Base de datos que utiliza el proyecto.
          Host: Dirección IP del servidor.
      Diccionario: Fichero índice.
      Usuario: Usuario de acceso a la BD.
      Password: Clave de acceso a la BD.

     CONFIGURA ENTORNO.
      Orden y texto de las opciones: idioma, tipo de desktop, color base.

     ELEMENTOS A GENERAR.
      Crear Database: Crear la base de datos donde se albergaran las tablas de la Intranet.
      Crear Tablas: Crear las tablas de la Intranet.
      Crear Ficheros de definición: Tomando como referencia la estructura de las tablas se crean los formularios asociados automáticamente.
      Crear Árbol de opciones: Crear las opciones de menú de la Intranet.
      Borrar directorios: Si se crea un proyecto igual que otro anterior borrar los directorios antiguos.

     GENERACIÓN DF.
      Wrap en Inputext: Permite hacer salto de línea en campos de texto normales.
      Tamaño del textarea: En el formulario de campos de tipo TEXT podemos definir el numero de caracteres a introducir ancho de la fila y numero de columnas.

     USUARIO.
      Password por defecto de la Aplicación: La clave que se le asigna a todos los usuarios que se dan de alta por defecto.
      Password de entrada con la Web parada: Cuando ejecutamos la opción de parar Web y necesitamos reactivarla nos pedirá una clave.

     APACHE.
      Ubicación del httpd.conf: ubicación del fichero de configuración del servicio Apache.
  • Definir Base de Datos

     Pantalla donde hay que introducir la estructura de la base de datos con el modelo definido en el apartado de AYUDA Definición SQL
  • Ver BD E-DES

     Muestra la estructura de las tablas que aporta el motor E-DES a la Intranet.
  • Cargar tabla auxiliar
     Cargar tabla de datos auxiliares
  • Importar tabla auxiliar
     Importar tabla de datos auxiliares 
  • Borrar tabla a importar
     Borrar tabla Importada de datos auxiliares 
  • Crear Intranet

     Proceso que coloca todos los ficheros de la Intranet en el dominio seleccionado y deja operativa la Intranet para su configuración y desarrollo final.
  • Bloquear Intranet
     Bloquear  la Intranet seleccionada, para empezar a trabajar directamente sobre los formulario creados.
  • Usuarios de desarrollo

     Pantalla de permisos de usuarios de desarrollo de la Intranet.
